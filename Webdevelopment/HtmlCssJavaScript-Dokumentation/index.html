Was ist HTML

HTML, oder HyperText Markup Language, ist eine Auszeichnungssprache, die verwendet wird, um Inhalte auf Webseiten zu strukturieren und anzuzeigen. Mit HTML können Entwickler Text, Bilder, Links, Tabellen und andere Elemente auf einer Webseite definieren. HTML-Dokumente bestehen aus Tags, die den Browser anweisen, wie der Inhalt angezeigt werden soll. Diese Tags sind in spitzen Klammern geschrieben, wie z.B. <p> für einen Absatz oder <a> für einen Link.

HTM ist keine eigenständige Technologie, sondern eine Dateierweiterung, die manchmal anstelle von HTML verwendet wird. Dateien mit der Erweiterung .htm sind funktionell identisch mit .html-Dateien und werden von Webbrowsern gleichermaßen behandelt. Der Unterschied liegt lediglich in der Namenskonvention, die historisch bedingt ist und auf alten Betriebssystemen wie MS-DOS zurückgeht, die nur drei Zeichen für Dateierweiterungen erlaubten.

Wie ist HTML aufgebaut

            
<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>Document</title>
    </head>
    <body>
    </body>
</html>

Das <!DOCTYPE>-Tag hat eine wichtige Funktion: Es teilt dem Webbrowser mit, dass es sich bei dem Dokument um eine HTML5-Datei handelt. Dies sorgt dafür, dass der Browser die neuesten HTML5-Standards und -Spezifikationen anwendet und somit die Webseite korrekt darstellt.

Der <html>-Tag umschließt den gesamten Inhalt des HTML-Dokuments. Das Attribut lang="de" gibt an, dass der Inhalt der Webseite in deutscher Sprache verfasst ist, was Suchmaschinen und Screenreadern hilft, die Seite richtig zu interpretieren.

Innerhalb des <head>-Tags befinden sich Metadaten und andere Informationen über das Dokument, die nicht direkt auf der Webseite angezeigt werden. Diese Informationen sind wichtig für die korrekte Funktion und Darstellung der Seite.

Das <meta charset="UTF-8">-Tag legt den Zeichensatz der Webseite fest. UTF-8 ist der am häufigsten verwendete Zeichensatz im Web, da er nahezu alle Schriftzeichen und Symbole unterstützt.

Das <meta name="viewport" content="width=device-width, initial-scale=1.0">-Tag sorgt dafür, dass die Webseite auf mobilen Geräten richtig dargestellt wird. Es passt die Breite des Ansichtsfensters an die Breite des Geräts an und stellt sicher, dass die Seite nicht skaliert wird.

Das <title>-Tag gibt den Titel der Webseite an, der in der Titelleiste des Browsers angezeigt wird. Dieser Titel ist auch wichtig für Suchmaschinen, da er in den Suchergebnissen angezeigt wird.

Der <body>-Tag enthält den gesamten sichtbaren Inhalt der Webseite, wie Texte, Bilder, Links und andere Elemente. Alles, was innerhalb des <body>-Tags steht, wird im Browser angezeigt.

Schließlich wird das Dokument mit dem schließenden </html>-Tag beendet, das den Beginn des <html>-Tags schließt und das Ende des HTML-Dokuments markiert.

HTML-Formularelemente ermöglichen Benutzern die Eingabe und Übermittlung von Daten an einen Server. Wenn ein Benutzer das Formular absendet, werden die eingegebenen Informationen gemäß der im form-Tag angegebenen action-URL und method (z.B. POST oder GET) an den Server übertragen. Jedes Formularelement wie input, select, textarea, radio, checkbox etc. hat ein name-Attribut, das als Schlüssel fungiert, und die eingegebene oder ausgewählte Information wird als Wert übermittelt. Diese Datenpaare (Schlüssel-Wert) werden in einem HTTP-Request an den Server gesendet. Vor der Übertragung werden die Daten vorübergehend im Arbeitsspeicher (RAM) des Geräts des Benutzers gespeichert. Der Server empfängt diese Daten, verarbeitet sie entsprechend der Serverlogik (z.B. Speicherung in einer Datenbank), und kann darauf basierend Antworten zurück an den Benutzer senden, wie z.B. Bestätigungen oder Fehlermeldungen.

CSS overflow

Die overflow-Eigenschaft in CSS bestimmt, was passieren soll, wenn der Inhalt eines Elements den Bereich des umgebenden Containers überschreitet. Dies ist besonders nützlich, um zu steuern, wie abgeschnittene oder scrollbar Inhalte dargestellt werden.

Die overflow-Eigenschaft kann folgende Werte annehmen:

  • visible: Der überfließende Inhalt wird nicht abgeschnitten und ist sichtbar (Standardwert).

  • hidden:: Der überfließende Inhalt wird abgeschnitten und ist nicht sichtbar.

  • scroll: Scrollleisten werden hinzugefügt, um den gesamten überfließenden Inhalt anzuzeigen, unabhängig davon, ob der Inhalt überläuft oder nicht.

  • auto: Scrollleisten werden nur hinzugefügt, wenn der Inhalt überläuft.

Einführung in JavaScript

Was ist JavaScript

JavaScript ist eine der am weitesten verbreiteten Programmiersprachen der Welt. Es wird hauptsächlich für die Entwicklung von Webanwendungen verwendet und läuft in nahezu jedem Webbrowser. JavaScript ist eine dynamische, schwach typisierte und interpretierte Sprache.

Typsicherheit in JavaScript

Typsicherheit bezieht sich auf die Art und Weise, wie eine Programmiersprache mit Datentypen umgeht und Fehler in Bezug auf falsche Datentypen verhindert. Sprachen können stark oder schwach typisiert sein.

Schwache Typisierung

JavaScript ist eine schwach typisierte Sprache. Das bedeutet, dass Variablen nicht an einen bestimmten Datentyp gebunden sind. Man kann einer Variable einen Wert eines beliebigen Datentyps zuweisen, und dieser Typ kann sich im Laufe des Programms ändern.

JavaScript als interpretierte Sprache

Ein Interpreter ist ein Programm, das den Quellcode Zeile für Zeile liest, analysiert und ausführt. JavaScript wird typischerweise in einem Webbrowser von einem JavaScript-Interpreter (z.B. V8 in Google Chrome, SpiderMonkey in Mozilla Firefox) ausgeführt.

Vorteile von interpretierten Sprachen

  • Einfachheit der Ausführung: Kein separater Kompilierungsschritt erforderlich. Der Quellcode wird direkt ausgeführt.

  • Flexibilität: Interpretierten Sprachen erlauben dynamische Änderungen und sofortige Ausführung des Codes.

Nachteile von interpretierten Sprachen

  • Leistung: In der Regel langsamer als kompilierte Sprachen, da der Code zur Laufzeit interpretiert wird.

  • Fehlererkennung: Fehler werden erst zur Laufzeit entdeckt, was das Debugging schwieriger machen kann.

Grundlegende JavaScript-Syntax

Einfache Ausgabe

console.log("Hello, World!");

Die Methode console.log() wird verwendet, um Nachrichten in die Konsole des Browsers zu schreiben. Es ist nützlich für Debugging-Zwecke und um Informationen während der Ausführung des Codes anzuzeigen.

Variablen

var name = "John";
let age = 30;
const isStudent = true;

JavaScript unterstützt verschiedene Arten von Variablen:

  • var: Deklariert eine Variable mit funktionalem oder globalem Scope.
  • let: Deklariert eine Variable mit Blockscope.
  • const: Deklariert eine Konstante mit Blockscope, deren Wert nicht geändert werden kann.

Verwenden von let anstelle von var

Die Verwendung von let gegenüber var in JavaScript bietet mehrere Vorteile, die hauptsächlich mit dem Scope-Verhalten und der Fehlervermeidung zu tun haben. Hier sind die wichtigsten Gründe:

Block-Scope vs. Function-Scope

var: Variablen, die mit var deklariert werden, haben einen function-scope. Das bedeutet, dass sie innerhalb der gesamten Funktion, in der sie deklariert wurden, verfügbar sind, unabhängig davon, in welchem Block sie sich befinden. 

                
    function example() {
        if (true) {
            var x = 10;
        }
        console.log(x); // 10
        }
        
        example();

let: Variablen, die mit let deklariert werden, haben einen block-scope. Das bedeutet, dass sie nur innerhalb des Blocks, in dem sie deklariert wurden, verfügbar sind. 

                
    function example() {
        if (true) {
            let x = 10;
            console.log(x); // 10
        }
        // console.log(x); // ReferenceError: x is not defined
        }
        
        example();

    let isTrue = false;

    let numbers = [1, 2, 3, 4, 5];

JavaScript unterstützt verschiedene Datentypen, einschließlich:

  • Primitive Datentypen: String, Number, Boolean, Null, Undefined, Symbol, BigInt.
  • Komplexe Datentypen: Object, Array, Function.

  • Vergleichsoperatoren: ==, ===, !=, !==, >, <, >=, <=
  • Logische Operatoren: &&, ||, !

Anonyme Funktionen und Arrow Functions

                
    // Anonyme Funktion
    const add = function(a, b) {
        return a + b;
    };

    // Arrow Function
    const multiply = (a, b) => a * b;

    console.log(add(2, 3));
    console.log(multiply(2, 3));

JavaScript unterstützt auch anonyme Funktionen und Arrow Functions, die kürzere Syntax für Funktionen bieten.

Objekte und Arrays

Objekte

                
    let person = {
        name: "John",
        age: 30,
        greet: function() 
        {
            console.log("Hello, " + this.name);
        }
    };
    console.log(person.name);
    person.greet();

Objekte sind Sammlungen von Schlüssel-Wert-Paaren.

Arrays

                
    let numbers = [1, 2, 3, 4, 5];

    for (let i = 0; i < numbers.length; i++) {
        console.log(numbers[i]);

    }

Arrays sind geordnete Listen von Werten.

DOM-Manipulation

Zugriff auf Elemente

                
    let element = document.getElementById("myElement");
    let elements = document.getElementsByClassName("myClass");
    let items = document.querySelectorAll(".myClass");

Mit JavaScript kannst du die Struktur, den Inhalt und das Aussehen einer Webseite dynamisch ändern.

Ändern von Inhalten

                
    element.innerHTML = "Neuer Inhalt";
    element.style.color = "red";

JavaScript ermöglicht das Ändern des HTML-Inhalts und der CSS-Stile von Elementen.

Ereignisse

                
    element.addEventListener("click", function() {
        alert("Element wurde geklickt!");
    });

JavaScript kann Ereignislistener zu Elementen hinzufügen, um auf Benutzerinteraktionen zu reagieren.

Asynchrone Programmierung

Callbacks

                
    function loadData(callback) {
        setTimeout(() => {
            callback("Daten geladen");
        }, 1000);
    }

    loadData((message) => {
        console.log(message);
    });

Callbacks sind Funktionen, die als Argument an andere Funktionen übergeben werden und nach Abschluss einer asynchronen Operation aufgerufen werden.

Promises

                
    let promise = new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve("Daten geladen");
        }, 1000);
    });

    promise.then((message) => {
        console.log(message);
    });

Promises sind Objekte, die eventual completion (oder failure) und den result einer asynchronen Operation darstellen.

Async/Await

                
    async function loadData() {
        let data = await new Promise((resolve) => {
            setTimeout(() => {
                resolve("Daten geladen");
            }, 1000);
        });

        console.log(data);
    }

    loadData();

Async/Await ist eine syntaktische Verbesserung für die Arbeit mit Promises, die es ermöglicht, asynchronen Code wie synchronen Code zu schreiben.

Klassen in JavaScript

JavaScript-Klassen sind Vorlagen für Objekte. Sie erleichtern die Erstellung und Verwaltung von Objekten und sind ein integraler Bestandteil moderner JavaScript-Entwicklung.

      
      class Person {
        constructor(name, age) {
          this.name = name;
          this.age = age;
        }
      
        greet() {
          console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
        }
      }
      
      const person = new Person('Alice', 30);
      person.greet(); // Ausgabe: Hello, my name is Alice and I am 30 years old.

Kapselung und Entkapselung

Kapselung ist das Konzept, die internen Details einer Klasse zu verstecken und nur die notwendigen Teile offenzulegen. In JavaScript können private Felder mit einem vorangestellten `#` gekennzeichnet werden.

      
      class Person {
        #name; // privates Feld
        #age;  // privates Feld
      
        constructor(name, age) {
          this.#name = name;
          this.#age = age;
        }
      
        greet() {
          console.log(`Hello, my name is ${this.#name} and I am ${this.#age} years old.`);
        }
      
        getName() {
          return this.#name;
        }
      
        setName(name) {
          this.#name = name;
        }
      }
      
      const person = new Person('Alice', 30);
      person.greet(); // Ausgabe: Hello, my name is Alice and I am 30 years old.
      console.log(person.getName()); // Ausgabe: Alice
      person.setName('Bob');
      console.log(person.getName()); // Ausgabe: Bob

Vererbung

Vererbung ermöglicht es, eine Klasse von einer anderen Klasse abzuleiten, sodass die abgeleitete Klasse die Eigenschaften und Methoden der Basisklasse erbt.

      
      class Animal {
        constructor(name) {
          this.name = name;
        }
      
        speak() {
          console.log(`${this.name} makes a noise.`);
        }
      }
      
      class Dog extends Animal {
        constructor(name, breed) {
          super(name);
          this.breed = breed;
        }
      
        speak() {
          console.log(`${this.name} barks.`);
        }
      }
      
      const myDog = new Dog('Rex', 'German Shepherd');
      myDog.speak(); // Ausgabe: Rex barks.

Statische Methoden

Statische Methoden sind direkt auf der Klasse selbst definiert und nicht auf den Instanzen der Klasse. Sie werden mit dem Schlüsselwort `static` definiert.

      
      class MathUtils {
        static add(a, b) {
          return a + b;
        }
      }
      
      console.log(MathUtils.add(5, 3)); // Ausgabe: 8

Overwrite

Method Overriding in JavaScript ermöglicht es, eine Methode in einer abgeleiteten Klasse mit demselben Namen wie in der Basisklasse zu definieren, wodurch die Methode der Basisklasse überschrieben wird.

      
      class Animal {
        speak() {
          console.log('Animal makes a noise.');
        }
      }
      
      class Dog extends Animal {
        speak() {
          console.log('Dog barks.');
        }
      }
      
      const myDog = new Dog();
      myDog.speak(); // Ausgabe: Dog barks.
